## 세부차수 구상

###  3-1: Cursor AI Rule

#### AI 어시스턴트 제어 및 커스터마이징

  - 특정 규칙, 스타일 가이드, 제약사항 등을 기술하는 파일입니다.

#### 신규 MDC 형식 도입

  - 기존 일반 텍스트에서 **Markdown 스타일**로 변경되었습니다.
  - old `.cursorrules` 형식은 deprecated 되었습니다.
  - `.cursor/rules` 디렉토리 밑에 mdc 파일을 생성해야 합니다.

#### 관련 자료

  - **오피셜 문서**: [https://docs.cursor.com/en/context/rules](https://docs.cursor.com/en/context/rules)
  - **참고 Github**: [https://github.com/PatrickJS/awesome-cursorrules](https://github.com/PatrickJS/awesome-cursorrules)
      - `rules-new` 폴더 내용을 주로 참조하면 됩니다.

####  4가지 참조 모드

  - **Always Apply**: 항상 적용
  - **Apply Intelligently**: 지능적으로 적용
  - **Apply to Specific Files**: 특정 파일에만 적용
  - **Apply Manually**: 수동으로 적용

### 3-2: Agent에 Context 정보 제공

#### LLM의 Context 활용

  - LLM은 텍스트, 이미지, 문서 등 **멀티모달 정보**를 Context로 받을 수 있습니다.
  - 이는 LLM의 **knowledge cut-off 한계**를 극복하는 데 도움이 됩니다.

####  Context 정보 제공 방법

1.  **Cursor Rules 사용**
2.  **Add doc**: 웹사이트 공식 문서를 직접 제공
3.  **파일 참조**: 문서가 클 경우, 직접 다운로드하여 파일로 참조
    ```bash
    npx sitefetch https://react-hook-form.com/get-started -o react-hook-form.md
    ```
4.  **Agent 창에서 `@` 활용**:
      - `filecode`: 파일 코드
      - `docs`: 문서
      - `git history`: Git 기록
      - `rules`: 설정된 규칙
      - `terminal`: 터미널 내용
      - `past chats`: 이전 대화 내용
5.  **Notepad 활용**:
      - 임시적인 내용을 작성하여 활용 가능합니다.
      - 내용이 규칙화된다면 **Cursor Rules**로 격상하여 재사용할 수 있습니다.

#### 시연 Git 프로젝트

  - **Next.js SaaS Starter**: [https://github.com/nextjs/saas-starter](https://github.com/nextjs/saas-starter)

###  3-3: Cursor에서 git 사용 (15분)

####  Git 작동 원리

  - 2005년 리누스 토발즈가 개발했습니다.
  - **스냅샷 방식**: 변경 사항만 저장하는 델타 방식과 달리 전체를 저장하여 속도가 빠릅니다.
  - ![[Pasted image 20250904175305.png]]
  - 대부분의 작업이 로컬에서 이루어져 가볍고 빠릅니다.

#### Github 계정 생성

#### Github 인증 방법

1.  **SSH Key**:
      - 영구적으로 사용 가능하며, `git pull`, `push`, `commit` 등 단순 작업에 널리 사용됩니다.
      - `ssh-keygen` 명령어로 생성 후, `cat ~/.ssh/id_rsa.pub` 내용을 복사하여 Github에 등록합니다.
      - profile -> settings -> SSH and GPG keys 에서 New SSH key
2.  **gh auth login**:
      - 웹 브라우저를 통한 로그인 방식입니다.
      - 만료 기간 및 권한 설정이 가능하며, `gh cli` 등 다양한 환경에서 사용됩니다.
3.  **Personal Access Token**:
      - API 호출로 Git을 사용해야 할 경우, 헤더에 토큰을 전달해야 합니다.
      - profile -> settings -> developer settings -> Personal access tokens -> File-grained tokens

#### Git 사용의 중요성

  - **작업 전 `commit`**: 큰 작업 전에는 커밋하는 습관을 들여야 합니다.
  - **타스크별 `commit`**: AI가 의도치 않게 코드를 망쳤을 경우, 언제든지 특정 시점으로 롤백이 가능합니다.

#### Cursor Git 시연

1.  Init repo (저장소 초기화)
2.  Commit 진행
3.  Publish branch (브랜치 발행)
4.  Github 페이지에서 신규 저장소 생성 과정 시연
5.  **saas-starter 클론**: [https://github.com/nextjs/saas-starter](https://github.com/nextjs/saas-starter)

#### VSCode Git 기능 탐험

  - **Open Github 메뉴**: Github 페이지 열기
  - **Cherry Pick 후 Conflict 해결**: Cherry pick 이후 발생하는 충돌을 LLM으로 해결
  - **Branch 히스토리**: 복잡한 브랜치 히스토리 관리
      - **추천 영상 (git flow)**: [https://www.youtube.com/watch?v=EV3FZ3cWBp8](https://www.youtube.com/watch?v=EV3FZ3cWBp8)

#### Git 롤백 (되돌리기)

1.  **Checkout**:
      - 특정 시점으로 잠시 돌아가 버그 등을 찾을 때 유용합니다. (HEAD만 이동)
      - GUI 에서도 그래프에서 오른쪽 버튼 후 `checkout ( detached )` 매뉴
      - `git checkout <commit_hash>`
      - `git checkout main` 명령어로 다시 최신 상태로 복귀합니다.
2.  **Reset Hard**:
      - 타임머신처럼 과거로 완전히 돌아가며, 그 이후의 기록은 사라집니다.
      - `git reset --hard <commit_hash>`: 특정 커밋으로 돌아감
      - `git reset --hard HEAD`: 최신 커밋 이후의 모든 변경사항 삭제

### 3-4: 기존 소스 온보딩 방법 (10분)

####  LLM을 활용한 문서 자동 생성

  - 전체 소스 코드를 기반으로 다양한 설계 문서를 생성할 수 있습니다.
      - **PRD (Product Requirements Document)**: 제품의 기능, 목적, 우선순위를 정리한 문서
      - **IA (Information Architecture)**: 서비스의 정보 구조를 시각적으로 설계한 문서
      - **UseCase**: 사용자가 제품을 사용하는 구체적인 시나리오를 설명한 문서

#### 분석 대상 프로젝트

  - **Next.js SaaS Starter**: [https://github.com/nextjs/saas-starter](https://github.com/nextjs/saas-starter)

#### 분석 과정

1.  **PRD, IA, UseCase 생성**
2.  **Flow Chart 생성**:
      - 유저의 흐름에 따른 작업 흐름을 소스 코드별로 표현합니다.
      - **Mermaid 차트**를 활용하기 위해 관련 문서와 확장을 설치합니다.
          - **문서**: [https://mermaid.js.org/intro/syntax-reference.html](https://mermaid.js.org/intro/syntax-reference.html)
          - **확장**: `markdown preview mermaid support`

#### Flow Chart 생성 프롬프트 예시

```
- what is the purpose of this project @전체폴더?
- what major depenencies does this project use?
- @flow.md @Mermaid Syntax create complete flow of my app
- can you improve my mermaid diagram and simlify for quick high-level understanding? @Mermaid Syntax
- update the nodes with matching filenames in my project. So I know exactly where I can make changes.
```
