## 2차시 상세 개요

### 2-1: Cursor AI 소개 (3분)

#### 소개: The AI Code Editor

  - **핵심 가치**: "build to make you extraordinarily productive" (믿을 수 없을 정도로 높은 생산성을 제공)
  - **정의**: AI를 사용하여 코딩하는 가장 좋은 방법이자, 강력한 **Coding Agent**입니다.
  - **배경**: LLM 모델의 성능이 향상되면서 코딩 생산성이 몇 배에서 몇십 배까지 증가하고 있습니다.

#### 유료 요금제

  - **요금제 종류**:
      - **Free**: 2주 무료 체험, 기본 모델(Slow mode) 사용 가능
      - **Pro**: 취미 개발자용
      - **Ultra**: 개발 전문가용
  - **유료 플랜 특징**:
      - GPT-4o 등 프리미엄 LLM 모델 사용 가능
      - **컨텍스트 사이즈**가 커서, 대규모 코드 수정이나 복잡한 계획 수립 시 성능 차이가 발생
  - *LLM 모델별 상세 특징과 가격은 자료실을 참조하세요.*

### 2-2: Cursor App 및 환경 설정

#### Cursor 다운로드

  - **다운로드 홈페이지**: [https://cursor.com/downloads](https://cursor.com/downloads)
  - **권장 OS**: **Mac** 사용이 편리합니다. (AI 시대의 디폴트, 대부분의 프로그램이 우선적으로 지원)

#### Mac 개발 환경 설정

1.  **Homebrew 설치**:
    ```bash
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
    ```
2.  **Git 및 Node.js 설치**:
    ```bash
    # Git 설치
    brew install git

    # Git 기본 설정 (이름과 이메일을 본인의 것으로 변경)
    git config --global user.name "Your Name"
    git config --global user.email "your.email@example.com"

    # Node.js 설치
    brew install node
    ```

#### Windows 개발 환경 설정

  - **WSL (Windows Subsystem for Linux)** 설치를 권장합니다.
  - *자세한 설치 방법 및 문제 해결 가이드는 별도 문서로 제공됩니다.*

### 2-3: Cursor 기본 인터페이스 및 주요 기능 (7분)

#### 기본 인터페이스

  - **프로젝트 열기**:
      - `Open Folder`: 기존 폴더 열기
      - `Clone Repo`: Git 저장소 복제하여 열기
  - **Pane (창) 설명**:
      - **중앙**: 코드 에디터
      - **왼쪽/오른쪽**: 파일 탐색기, 확장 기능 등
      - **하단**: 터미널 (`cmd + j` 로 열기/닫기)
  - **Extensions (확장 프로그램)**:
      - VS Code와 완벽히 호환되어 모든 확장 프로그램 사용 가능
      - (실습) **Live Server** 확장 설치

#### Chat Mode

  - **Agent Mode**: AI가 직접 코드를 수정하고 작업을 수행하는 모드
  - **Ask Mode**: 코드 수정 없이 질문에 대한 답변만 받는 모드

#### Prompt Bar ( `cmd + k` )

  - **Edit Selection**: 선택한 코드 블록 수정
  - **Quick Question**: 간단한 질문
  - **Edit Full File**: 파일 전체 수정

#### 주요 단축키

  - `cmd + shift + p`: 모든 명령어 표시 (Show all commands)
  - `cmd + p`: 파일 탐색기 (File explorer)
  - `cmd + /`: 주석 처리 (Comment)
  - `cmd + k`: 프롬프트 바 열기 (Prompt bar)
  - `cmd + l`: AI 채팅 창 열기 (Agent window)
  - `cmd + j`: 터미널 열기/닫기 (Terminal)
  - `cmd + b`: 왼쪽 창 열기/닫기 (Toggle left pane)

#### Settings (설정)

  - Cursor 설정은 `Cursor Settings`와 `VS Code Settings`로 나뉩니다. (주로 `Cursor Settings`를 다룹니다.)
  - **General**:
      - `Completion Sound`: AI 작업 완료 시 소리 알림
  - **Chat**:
      - `Context Include Full-folder Context`: Agent에게 전체 폴더 정보를 컨텍스트로 제공 (필수 체크)
      - `Auto-Run Mode`: AI가 제안하는 작업을 자동으로 실행할지 여부 설정
	      - Ask Everything : 모든 것을 다 물어봄 
	      - Use Allowlist : 보수적으로 접근하는 분 
	      - Run Everything  : 추천 
      - `File-Deletion Protection`: 파일 자동 삭제 방지
  - **Indexing & Docs**:
      - `Codebase Indexing`: Agent가 프로젝트 전체 구조를 파악하기 위해 코드베이스를 인덱싱합니다. (소스는 로컬에 유지, 임베딩 데이터와 메타데이터만 클라우드에서 분석)
      - `Ignore Files in .cursorignore`: 인덱싱에서 제외할 파일을 지정합니다.
      - `Docs`: React, Next.js 등 특정 기술의 공식 문서를 인덱싱하여 Agent의 정확도를 높입니다.

### 2-4: Cursor AI TAB 완성 실습

  - TAB 키를 이용한 자동 완성 기능으로 간단한 레시피 웹사이트를 구현합니다.

#### 준비 과정

1.  **확장 프로그램 설치**: `Live Server`, `Prettier` 설치
2.  **데이터 파일 생성 (`recipes.js`)**:
      - Prompt Bar (cmd+k) 로  10개의 레시피 샘플 데이터 요청
        ```
        give me 10 samples of recipes, include ingredients, title, nutiritions, instructions, duration, image
        ```
      - 코드가 생성되면 `Prettier`로 포맷 정리 (`Format Document`)
      - `export { recipes }` 코드를 추가하여 데이터를 내보낼 수 있게 설정

#### 웹사이트 구현

1.  **HTML 파일 생성 (`index.html`)**: `!` 입력 후 `Tab` 키를 눌러 기본 구조 생성
2.  **스크립트 파일 생성 (`script.js`)**:
      - `recipes.js`에서 데이터를 `import`
      -  `appendChild` 키워드가 포함되었는지 확인 
            - 실제 레시피 카드들을 화면에 추가
      - **에러 해결**: 브라우저 개발자 도구(`option + cmd + j`)를 열어 콘솔 에러를 확인
      - Agent Mode를 통해 AI에게 에러 해결 요청
3.  **디자인 개선**:
      - AI에게 아래 프롬프트로 시각적인 개선 요청
            - `모던한 UX로 recipe 요소까지 개선해주세요`
4.  **설정 변경 (선택 사항)**:
      - VS Code 설정에서 `editor.acceptSuggestionOnCommitCharacter`를 `off`로 변경하면, 특정 문자 입력 시 자동 완성이 트리거되는 것을 방지할 수 있습니다.
